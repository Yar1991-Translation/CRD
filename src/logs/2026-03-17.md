## Added

- 新增 `src/logs/` 目录，作为后续更新日志的统一维护位置。
- 新增按日期命名的日志文件机制，例如 `2026-03-17.md`。
- 新增更新日志索引层，用于自动收集并排序日志文件。
- 新增日志语法说明文档 `SYNTAX.md`。
- 新增专用于 CRD 更新日志的自定义标签体系。

## Changed

- 更新历史页改为从日志文件中读取内容，而不是继续使用硬编码页面内容。
- 更新历史页左侧增加日期切换区，支持按日期查看不同日志。
- 日志正文区改成动态渲染结构，后续新增日志不需要改页面组件。
- 语法示例区改为读取独立的语法说明文件。

## Fixed

- 修复“查看更新日志语法示例”打开后显示成正式更新日志的问题。
- 修复日志 front matter 与部分列表内容被挤在同一行，导致摘要与内容解析不正确的问题。
- 修复只有一篇日志时不容易验证日期切换的问题，现在补充了另一篇日期日志用于展示。

## Docs

- 现在推荐所有新日志都按日期单独写入 `src/logs/`。
- 建议每篇日志都包含 front matter，用于日期列表和摘要展示。
- 建议在正式更新说明中混用 Markdown 和 CRD 专属标签，以兼顾可读性和视觉表达。

<crd-log-callout title="后续维护方式" tone="primary">
  <ol>
    <li>在 <code>src/logs/</code> 下新建一个 <code>YYYY-MM-DD.md</code></li>
    <li>写入 front matter、Markdown 内容和自定义标签</li>
    <li>页面会自动把新日志加入日期切换列表</li>
